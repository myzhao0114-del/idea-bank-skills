# 单篇论文精读笔记模板

为每篇论文复制以下结构，填充内容。括号内文字为填写提示，写完后删除提示文字。

```markdown
# 《论文标题》精读笔记

- 作者/年份/出处：
- 一句话总结（如果我是作者，我会怎么向同行推销这个想法）：

## 1. 设计思想（Why）

- 目标问题/任务：
- 现有方法的局限（作者认为"哪里不够"）：
- 核心洞见/假设（这个洞见为什么能解决上述局限）：
- 洞见 → 设计选择的对应关系（例如："假设空间自相关存在" → "引入邻域聚合模块"）：

## 2. 方法拆解（What / How）

### 2.1 整体流程

按论文实际顺序列出每一步，格式：步骤名 — 在做什么 — 为什么需要这一步

1. ...
2. ...
3. ...

### 2.2 核心模型/算法

- 结构描述（用自己的话，避免逐条堆砌公式）：
- 关键公式（挑 1-3 个最核心的，逐符号解释直觉）：
- 关键超参数：

### 2.3 消融实验结论（如有）

- 哪些组件是核心（去掉会显著掉点）：
- 哪些组件是锦上添花（去掉影响不大）：

## 3. 输入数据

| 项目 | 内容 |
|---|---|
| 数据类型 | 栅格影像 / 格网提取的表格 / 矢量 / 点位观测 / 时间序列 / 图结构 / 文本 / 图像 / 其他 |
| 空间分辨率 | |
| 时间分辨率/范围 | |
| 数据来源 | （数据集名称、传感器、统计年鉴、API 等） |
| 预处理/特征工程 | （归一化、重采样、裁剪、特征构造方式——具体到能照做的程度） |
| 训练/测试划分 | |
| 论文未说明的部分 | （明确标注，不要臆造） |

## 4. 输出结果

| 项目 | 内容 |
|---|---|
| 输出形态 | 预测图/栅格 / 分类标签 / 概率图 / 表格 / 排名 / 嵌入向量 / 其他 |
| 评价指标 | （列出指标名称，并说明为什么选这些指标） |
| 中间产物 | （是否产出可复用的中间表征，如特征图/分区结果/嵌入） |

## 5. 复现要点

- 用到的工具/库/平台：
- 计算资源量级（如论文有提及）：
- 代码/数据是否公开：

## 6. 跨领域迁移分析

- 领域无关的核心机制（可原样保留）：
- 领域特定、必须替换的部分：
- 原方法 → 目标场景对应表：

| 维度 | 原论文 | 迁移到用户场景 |
|---|---|---|
| 输入数据 | | |
| 关键特征/表征 | | |
| 模型/算法核心 | | |
| 输出 | | |
| 评价指标 | | |

- 迁移风险点（统计假设是否成立、样本量/分辨率差异等）：
- 具体改造建议（分步骤，可执行）：
```
