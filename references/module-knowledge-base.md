# 方法模块知识库：格式、分类与合并规则

`方法模块知识库.md` 是一个跨论文持续累积的文件。每读一篇新论文，就把其中"换一个数据集/领域依然成立的设计思路"提炼成模块，追加或合并进去。目标是：读的论文越多，越能一眼看出"哪些设计思路反复出现、可以直接搬到新任务上"。

## 文件骨架

新建文件时使用以下骨架：

```markdown
# 方法模块知识库

> 跨论文持续累积的可复用方法模块库。每次精读新论文后，提炼其中领域无关的设计思路，追加或合并到本文件。

## 模块索引

| 模块名 | 类别 | 解决的问题 | 适用数据形态 | 出现的论文 |
|---|---|---|---|---|

## 模块详情
```

## 模块条目格式

每个模块在"模块详情"下用三级标题，格式如下：

```markdown
### [类别] 模块名

- **解决的问题/动机**：这个模块是为了解决什么困境而存在的，不绑定具体论文的场景描述。
- **核心机制**：领域无关的原理性描述——讲清楚"为什么有效"，而不是某篇论文的具体超参数（层数、通道数等属于"实现细节"，放在"出现的论文"里）。
- **适用数据形态**：栅格/格网张量、表格、图结构、点云、时间序列等，以及方法对数据的隐含假设（如要求规则网格、平稳性、空间自相关等）。
- **跨领域应用提示**：用到别的数据/领域时，通常需要替换或适配的部分。
- **出现的论文**：
  - 《论文标题A》（年份）：该论文中的具体实现方式 + 与本模块"标准描述"的差异/变体。
  - 《论文标题B》（年份）：……
- **相关模块**：`[[模块名X]]`、`[[模块名Y]]`（链接到本文件中其他相关模块，模块不存在也可以先写上，作为以后补充的标记）。
```

## 推荐分类（不够用时可自行新增）

- 数据预处理/特征工程
- 时空表征/编码架构
- 注意力/交互机制
- 多步预测/误差控制策略
- 损失函数/训练目标
- 评价指标设计
- 输出后处理/不确定性量化

## 去重与合并规则

1. 新增模块前，先通读已有的"模块索引"表和相关分类下的条目。
2. 判断标准是**核心机制是否本质相同**，而不是论文用词是否相同（不同领域的论文常给同一机制起不同名字）。
3. 若已存在高度相似的模块：
   - 不新建条目。
   - 在该模块的"出现的论文"列表追加一行，写明本篇论文的具体实现和与已有描述的差异（例如分辨率、超参数、变体结构）。
   - 如果本篇论文的实现揭示了该机制更通用/更精确的描述方式，更新"核心机制"段落的措辞，使其覆盖所有已记录的论文。
4. 若是新机制，按模板新建条目，并在索引表追加一行。
5. 每次更新后检查索引表与详情是否一致（模块名、分类、出现论文列表）。

## 示例（基于已精读的 AI-GAMFS 论文）

以下是首次创建知识库、且只读过一篇论文（AI-GAMFS, *Nature*, 2026）时的示例内容，展示模块应该写到多"抽象"、多"领域无关"：

```markdown
## 模块索引

| 模块名 | 类别 | 解决的问题 | 适用数据形态 | 出现的论文 |
|---|---|---|---|---|
| ViT+U-Net 时空状态转移骨架 | 时空表征/编码架构 | 在规则网格上学习"当前时刻场 → 下一时刻场"的映射，兼顾全局依赖与空间细节 | 规则栅格/格网时空张量 | AI-GAMFS (2026) |
| 多变量联合嵌入 | 数据预处理/特征工程 | 让相互耦合的多组变量在模型内部互相提供信息 | 多通道栅格/表格，要求各通道时空覆盖一致 | AI-GAMFS (2026) |
| 周期性时间编码 | 数据预处理/特征工程 | 让模型感知时间的周期性（日/周/年） | 任意带时间戳的时序数据 | AI-GAMFS (2026) |
| 多提前期模型接力预报 | 多步预测/误差控制策略 | 缓解自回归多步预测的误差累积，同时不牺牲时间分辨率 | 任意需要多步迭代预测的时空/时序任务 | AI-GAMFS (2026) |

## 模块详情

### [时空表征/编码架构] ViT+U-Net 时空状态转移骨架

- **解决的问题/动机**：在规则网格上预测"下一时刻的场"时，既需要捕捉远距离的全局关联（如大气环流的远程传输），又需要保留局部空间细节（避免预测结果过度平滑）。
- **核心机制**：编码器逐级下采样提取多尺度空间特征；在最低分辨率上用全局自注意力（Transformer）建模长程/跨变量依赖；解码器逐级上采样并通过 skip connection 融合编码器对应层特征，恢复空间细节。本质是"全局注意力负责长程关系，卷积式编解码负责空间精度"的分工。
- **适用数据形态**：规则经纬度/笛卡尔网格上的多通道时空张量 (T×C×H×W)。要求空间结构连续、可分块（patch化）。
- **跨领域应用提示**：若目标数据不是规则网格（如格网提取后的不规则点表），需先重建为规则张量，或将卷积下采样/上采样替换为基于空间邻接关系的图网络，同时保留中心的全局注意力模块。全局注意力的计算量随 patch 数平方增长，分辨率很高时需改用窗口/局部注意力。
- **出现的论文**：
  - 《Advancing operational global aerosol forecasting with machine learning》（2026）：cube embedding（patch+线性投影+位置编码）→ 14 层 ViT → cube unembedding（反卷积+上采样），3 级下采样（1/2, 1/4, 1/8），约 1.2B 参数。
- **相关模块**：`[[多变量联合嵌入]]`、`[[多提前期模型接力预报]]`

### [数据预处理/特征工程] 多变量联合嵌入

- **解决的问题/动机**：当目标变量与其驱动变量存在强耦合关系时（例如气溶胶浓度受气象场调制），分开建模会丢失变量间的相互信息。
- **核心机制**：把目标变量和驱动变量打包进同一个多通道张量，一起输入同一个模型联合预测，让模型通过注意力/卷积自行学习跨变量的依赖关系，而不是人工指定哪个变量影响哪个变量。
- **适用数据形态**：多通道栅格或多列表格，要求各变量在训练期间时空覆盖一致（缺测会破坏联合建模的前提）。
- **跨领域应用提示**：迁移到新领域时，先确认"目标变量 + 候选驱动变量"是否能拿到时空对齐的同源数据；如果驱动变量缺测严重，需要先做缺测填补或退化为单变量建模。
- **出现的论文**：
  - 《Advancing operational global aerosol forecasting with machine learning》（2026）：12 个气溶胶变量 + 6 个地面气象变量 + 4 个高空气象变量×9 层，共 54 通道联合预测；论文报告气象预报精度提升直接带动了气溶胶预报精度提升。
- **相关模块**：`[[ViT+U-Net 时空状态转移骨架]]`

### [数据预处理/特征工程] 周期性时间编码

- **解决的问题/动机**：很多时空过程具有日周期、周周期、年周期等规律，但原始时间戳（如 UTC 时刻）本身不是模型容易利用的特征。
- **核心机制**：将时间戳分解为"一天中的小时、一周中的第几天、一年中的第几天"等周期性分量，编码后作为额外输入特征/通道，让模型直接感知周期信息，而不必从数据中隐式学习。
- **适用数据形态**：任意带时间戳的时序/时空数据。
- **跨领域应用提示**：通用性很强，几乎可以直接套用；需要根据目标领域的主导周期调整分量（例如农业遥感可能更关心"物候期/年内日序"而非"小时"）。
- **出现的论文**：
  - 《Advancing operational global aerosol forecasting with machine learning》（2026）：hour of day / day of week / day of year 三个分量。
- **相关模块**：`[[多变量联合嵌入]]`

### [多步预测/误差控制策略] 多提前期模型接力预报

- **解决的问题/动机**：自回归多步预测中，每一步的预测误差会累积放大，提前期越长误差越大；但提高时间分辨率（更小步长）会让所需迭代次数增多，进一步加剧误差累积。
- **核心机制**：训练若干个结构相同、但分别针对不同提前期（如 3h/6h/9h/12h）的"专用"基础模型；预测任意目标提前期时，优先选用提前期最大的模型做"接力"，以最少的迭代次数拼出目标提前期，从而在不牺牲时间分辨率的前提下减少误差累积。需注意：增加基础模型数量带来的收益边际递减。
- **适用数据形态**：任意需要多步迭代预测的时空/时序任务，与具体的骨架网络解耦（可以套在任何单步预测模型外面）。
- **跨领域应用提示**：迁移时先评估"单模型多步迭代"的误差累积是否真的是瓶颈（可通过误差随提前期增长曲线判断）；若是，再决定训练几个不同步长的基础模型——论文的消融显示边际收益递减，不必盲目增加。
- **出现的论文**：
  - 《Advancing operational global aerosol forecasting with machine learning》（2026）：4 个基础模型（3h/6h/9h/12h），对 >24h 提前期的预测，4 模型接力相比单一 3h 模型迭代，120h 提前期 RMSE 平均降低 15.1%；该思路借鉴自 Pangu-Weather 的时间聚合策略。
- **相关模块**：`[[ViT+U-Net 时空状态转移骨架]]`
```

后续读到新论文时，例如读到一篇"用 GNN 处理不规则监测站点 + 类似的多步接力预报策略"的论文：
- "多提前期模型接力预报"模块**不需要新建**——在其"出现的论文"里追加一行，并在"跨领域应用提示"或"核心机制"中补充"该机制与具体骨架网络无关，已在 ViT+U-Net 和 GNN 骨架上都得到验证"。
- 但"GNN 处理不规则站点"这个表征架构是新机制，应在"时空表征/编码架构"分类下新建一个独立模块，并在"相关模块"中与"ViT+U-Net 时空状态转移骨架"互相链接（说明二者是"规则网格 vs 不规则站点"的对应关系）。
